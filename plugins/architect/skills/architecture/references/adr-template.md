# ADR Template: Architecture Decision Record

For each significant architectural decision, document using the following structure:

## Status
- [ ] Proposed
- [ ] Accepted
- [ ] Deprecated
- [ ] Superseded

## Context
What is the issue, motivation, or decision being made? Provide background info and current state.

## Decision
What is the change being made? Explain the rationale and how it addresses the context.

## Consequences
- **Positive**: What are the benefits?
- **Negative**: What are the drawbacks?
- **Neutral**: What are the trade-offs?

## Alternatives
- **Option 1**: Description + why rejected.
- **Option 2**: Description + why rejected.

## Implementation
- [ ] Tasks to implement this decision
- [ ] Timeline
- [ ] Owner
