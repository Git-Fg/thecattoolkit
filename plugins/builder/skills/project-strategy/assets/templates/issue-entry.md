# Issue Entry Template

**MANDATORY:** Use this format when logging issues to ISSUES.md.

## Issue Entry Format

```markdown
# Issue: {ISSUE_TITLE}

## Status

{OPEN/IN_PROGRESS/RESOLVED/DEFERRED}

## Priority

{HIGH/MEDIUM/LOW}

## Description

{Detailed description of the issue or enhancement}

## Rationale

{Why this issue is important or why the enhancement is needed}

## Proposed Solution

{Brief description of proposed solution}

## Tasks

- [ ] Task 1: {DESCRIPTION}
- [ ] Task 2: {DESCRIPTION}

## Acceptance Criteria

- [ ] {CRITERION_1}
- [ ] {CRITERION_2}

## Notes

{Additional context or considerations}

## Related

- **Project:** {PROJECT_NAME}
- **Created:** {DATE}
- **Assigned To:** {ASSIGNEE}
```

## MANDATORY Rules

- **MUST** include discovery context (Phase/Plan/Task where found)
- **MUST** specify impact and effort estimate
- **MUST** include suggested phase for resolution
- **MUST** be verifiable (acceptance criteria must be testable)
