# Task Plan Template

**Purpose:** Track phases, progress, decisions, and errors for simple tasks (3-5 phases)

## Schema

```markdown
# Task Plan: {project-name}

## Objective
{One-sentence description}

## Progress Tracking

### Phase 1: {name}
- **Status:** [ ] Pending | [~] In Progress | [x] Complete
- **Tasks Completed:**
  - [ ] {task 1}
  - [ ] {task 2}
- **Notes:** {Observations, decisions, blockers}

### Phase 2: {name}
- **Status:** [ ] Pending | [~] In Progress | [x] Complete
- **Tasks Completed:**
  - [ ] {task 1}
  - [ ] {task 2}
- **Notes:** {Observations, decisions, blockers}

### Phase 3: {name}
- **Status:** [ ] Pending | [~] In Progress | [x] Complete
- **Tasks Completed:**
  - [ ] {task 1}
  - [ ] {task 2}
- **Notes:** {Observations, decisions, blockers}

## Decisions Log
| Date | Decision | Rationale |
|------|----------|-----------|
| {date} | {What was decided} | {Why this choice} |

## Errors & Resolutions
| Date | Error | Resolution |
|------|-------|------------|
| {date} | {What failed} | {How it was fixed} |

## Final Status
**Overall Progress:** {X}/{Y} phases complete
**Deliverable:** {Final output}
```

## Usage Guidelines
- Use for simple projects (3-5 phases)
- Update status codes in real-time
- Log all significant decisions
- Document errors and their resolutions
- Keep concise and actionable
